# annotations-design-patterns — Índice de subplanes

> **Para agentes de IA:** Este proyecto se implementa mediante 5 subplanes independientes,
> pensados para ser ejecutados por un agente por subplan, **en orden estricto**.
> Cada subplan es autocontenido: no necesitas leer los demás para ejecutar el tuyo,
> pero SÍ debes leer la especificación primero.

**Especificación:** `docs/superpowers/specs/2026-07-03-anotations-design-patterns-design.md`

## Skills que debe usar el agente ejecutor

Estas skills EXISTEN en este entorno y se invocan con la herramienta Skill:

- **`test-driven-development`** — invócala ANTES de implementar cada patrón. Todos los
  subplanes siguen TDD: test que falla → implementación mínima → test en verde → commit.
- **`typescript-best-practices`** — invócala al empezar cada subplan. Reglas clave del
  proyecto: `strict: true`, prohibido `any` salvo en la firma convencional de
  constructores `new (...args: any[]) => object` (única excepción permitida).
- **`typescript-wizard`** — solo necesaria en Fase 3 (tipos del builder) y Fase 4
  (genéricos avanzados). No la cargues en fases 0-2.
- **`verify`** — invócala al FINAL de cada subplan, antes del commit de cierre.

Skills NO instaladas (no intentes invocarlas): `superpowers:subagent-driven-development`
y `superpowers:executing-plans` no existen en este entorno. Los subplanes usan checkboxes
`- [ ]` y se ejecutan directamente, paso a paso, en orden.

## Reglas globales para todos los subplanes

1. **No improvises API.** El código de cada paso es el contrato: nombres, firmas y
   mensajes de error se copian tal cual.
2. **Decoradores TC39 únicamente.** PROHIBIDO añadir `experimentalDecorators` o
   `emitDecoratorMetadata` al tsconfig. Si un test falla con decoradores, el problema
   es otro: no toques esos flags.
3. **Cero dependencias de runtime.** Solo devDependencies. Si crees necesitar una
   dependencia, detente y pregunta al usuario.
4. **Un commit por patrón** (test + implementación + export en `index.ts` juntos).
5. **Cada patrón vive en su propio archivo** `src/patterns/<categoria>/<nombre>.ts`
   (categorías: `creational/`, `structural/`, `behavioral/`, `utility/`; reorganizado
   tras la v1 — los planes de las fases citan las rutas planas antiguas) y se
   re-exporta desde `src/index.ts`. No hace falta tocar `package.json` al añadir
   patrones: el subpath export `"./*"` cubre también rutas anidadas.
6. Mensajes de error de la librería en español, código y nombres de API en inglés.
7. **No actualices las devDependencies.** Los pines `typescript@^5.9` y `vite@^7` son
   deliberados (descubiertos en la Fase 0): TypeScript 6.x rompe el build de tipos de
   tsup (`TS5101: baseUrl deprecated`) y vite 8 (transformador oxc/Rolldown) todavía
   no downlevela decoradores TC39. No ejecutes `npm update` ni cambies esas versiones.
