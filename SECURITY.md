# Security Policy

## Supported Versions

Security fixes are considered for the latest published major version.

## Reporting A Vulnerability

Please do not open a public issue for security reports.

Report vulnerabilities by emailing the package author listed in `package.json`.
Include:

- affected version;
- reproduction steps or proof of concept;
- impact summary;
- suggested fix, if known.

We will acknowledge valid reports and coordinate a fix before public disclosure.
